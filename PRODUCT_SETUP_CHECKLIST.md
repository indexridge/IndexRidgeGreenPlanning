# Product Setup Checklist

## Phase 1 — validation launch

- [x] Create local project folder.
- [x] Add risk/copyright policy.
- [x] Register first open-data source and licence.
- [x] Build official-open-data sample fetcher.
- [x] Generate sample CSV, JSON, and Markdown digest.
- [x] Draft local landing page.
- [x] Publish free public beta page with no checkout, no analytics, no signup, and no outreach.
- [ ] Review sample records manually for commercial usefulness.
- [ ] Add at least 2 more approved open/public sources before selling coverage.
- [ ] Decide brand name, price, and platform.

## Phase 2 — pre-launch, still no public sales

- [ ] Build non-public sample issue from the latest records.
- [ ] Create source register with licence/terms notes for every source.
- [ ] Prepare Gumroad/Payhip/Stripe product draft but do not publish.
- [ ] Prepare support email and refund/cancellation wording.
- [ ] Prepare a small target-customer list using compliant, non-invasive sourcing.
- [ ] User approves exact public page and checkout copy.

## Phase 3 — paid launch only after approval

- [x] Publish free validation landing page.
- [ ] Publish payment product.
- [ ] Send only approved outreach.
- [ ] Track revenue, conversion, support, unsubscribes, and refunds.
- [ ] Reassess legal/data-source risk monthly.

## API keys/accounts likely needed later

- Payment platform: Gumroad, Payhip, Stripe, or Lemon Squeezy.
- Email platform: Buttondown, MailerLite, ConvertKit, Beehiiv, or a transactional email provider.
- Optional enrichment APIs only if their licence/terms allow commercial reuse.

## Do-not-do list

- Do not use copied articles/images/maps/screenshots.
- Do not resell proprietary competitor data.
- Do not imply official government affiliation.
- Do not claim complete UK coverage until proven.
- Do not provide planning/legal/investment advice.
- Do not send bulk unsolicited marketing without compliance review.
